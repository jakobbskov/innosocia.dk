# Deployment contract – innosocia.dk

## Purpose

This document defines how innosocia.dk should be built, deployed and validated.

The goal is a reversible, Git-first deploy process. The live server is not a workspace.

## Current known state

As of 2026-04-26:

- Repository: jakobbskov/innosocia.dk
- Local source of truth: /home/jakob/github/innosocia.dk
- Framework: Astro
- Output mode: static
- Build output: dist/
- Production URL configured in Astro: https://innosocia.dk
- Existing live web root: not yet confirmed
- Existing nginx vhost for innosocia.dk: not yet confirmed

## Required validation

From the repository root:

    node -v
    npm -v
    npm ci
    npm audit
    npm run build

Expected:

- Node.js is >=22.12.0
- npm audit reports 0 vulnerabilities
- Astro build completes successfully
- dist/index.html, dist/robots.txt and dist/sitemap-index.xml exist

## Proposed deploy model

The site should be deployed as static files from dist/ to a dedicated web root, for example:

    /var/www/innosocia.dk/

The deploy command should copy only generated static files:

    sudo mkdir -p /var/www/innosocia.dk
    sudo rsync -av --delete dist/ /var/www/innosocia.dk/

This must only be run after a successful local build.

## Proposed nginx contract

The nginx vhost should serve innosocia.dk and optionally www.innosocia.dk from the static web root.

Expected behavior:

- HTTP redirects to HTTPS
- HTTPS serves static Astro output
- unknown routes are handled as static files generated by Astro, not by a backend
- no backend service is required

Example nginx shape, adapted to actual host/container setup:

    server {
        listen 80;
        server_name innosocia.dk www.innosocia.dk;
        return 301 https://innosocia.dk$request_uri;
    }

    server {
        listen 443 ssl http2;
        server_name innosocia.dk www.innosocia.dk;

        root /var/www/innosocia.dk;
        index index.html;

        location / {
            try_files $uri $uri/ =404;
        }
    }

Certificate paths, proxy-container placement and reload commands must be adapted to the actual host setup.

## Live validation

After deploy:

    curl -I https://innosocia.dk/
    curl -I https://innosocia.dk/robots.txt
    curl -I https://innosocia.dk/sitemap-index.xml
    curl -I https://innosocia.dk/apps/
    curl -I https://innosocia.dk/status/

Expected:

- homepage returns 200
- robots.txt returns 200
- sitemap-index.xml returns 200
- generated routes return 200
- no 5xx responses

## Rollback

Rollback must be file-based and Git-based:

1. identify the previous known-good Git commit
2. check it out locally
3. run validation
4. rebuild
5. redeploy dist/

Do not debug by editing live files directly.
